# Contributing to HomeESS

First of all, thank you for your interest in contributing to HomeESS!

Contributions of any size are welcome, whether they are bug fixes, new features, documentation improvements, or performance optimizations.

## Before You Start

Please check existing Issues and Pull Requests before starting work on a larger feature.

For major changes, it is recommended to open an Issue first to discuss your proposal.

## Coding Guidelines

Please follow these basic guidelines:

- Keep changes focused and easy to review.
- Follow the existing coding style.
- Write clear and meaningful commit messages.
- Update documentation where appropriate.
- Avoid unnecessary dependencies.

## Pull Requests

Before opening a Pull Request, please ensure that:

- your code builds successfully;
- existing functionality is not broken;
- new functionality is documented where appropriate;
- the Pull Request focuses on a single topic whenever possible.

## Contributor License Agreement (CLA)

By submitting a Pull Request, you confirm that:

- you are legally entitled to submit your contribution;
- you have read and agree to the project's `CLA.md`;
- you grant the rights described in the CLA.

Pull Requests cannot be merged unless these conditions are met.

## Code of Conduct

Please be respectful and constructive.

Technical discussions are encouraged.

Personal attacks, harassment, discrimination, or abusive behavior will not be tolerated.

## Questions

If you are unsure about an implementation or would like feedback before investing significant time, feel free to open an Issue for discussion.

Thank you for contributing to HomeESS!
